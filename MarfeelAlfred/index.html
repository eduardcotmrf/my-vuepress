# MarfeelAlfred

# To commit

As we use Commitizen you can do either:

npx git-cz

Or

npm run commit

# To run

In order to run it we need to do: npm run start(?:dev) PUBLISHER_ID EXTRACTOR_NAME NUM_SECTIONS NUM_ARTICLES DEVICE_TYPE SOURCE BUSINESS_MODEL where:

  • PUBLISHER_ID: for instance elconfidencial.com
  • EXTRACTOR_NAME: for instance inventory
  • NUM_SECTIONS: Number of sections to visit (excluding home section). By default 0
  • NUM_ARTICLES: Number of articles to visit. By default 0
  • DEVICE_TYPE: Type of device: mobile, desktop or marfeel
  • SOURCE: Source or by default PRESS
  • BUSINESS_MODEL: Business Model of by default ENTERPRISE

e.g:

# in dev mode
npm run start:dev www.goalcast.com inventory 1 0 desktop

# in prod mode
npm run start www.goalcast.com inventory 1 0 desktop

We can also run it directly with node:

# in dev mode
NODE_ENV=dev node ./src/alfred "www.goalcast.com" "inventory" "1" "0" "desktop" "INSIGHT" "OAP"

# in prod mode
NODE_ENV=prod node ./src/alfred "www.goalcast.com" "inventory" "1" "0" "desktop" "INSIGHT" "OAP"

# To test

# run test matching the patter and watch
npm test -- --t blackList
# or
npm run test blackList

# run test for all
npm run test

Currently there's a bug with the watcher in OS X In order to make it work you need to do:

$ brew update
$ brew install watchman

# Fixtures

In order to create fixtures to run with your test you have to run:

npm run fixture <uri of your article/section/mosaic>

This will create a file in tests/fixtures/ with the wanted html

# Commands

# To install all dependencies
npm i

# To validate (lint and tests)
npm run validate


# To lint all files
npm run lint


# To run all the tests
npm run test


# To run the test with a watcher
npm run test:watch

# Hooks

Before you commit any change it will validate all:

eslint, tests

Before push your changes it will run all the test.

tests
